Guided Mode

The Guided Mode is intended to guide users step-by-step through all the requirements for curating and sharing datasets according to the SPARC data standards. The user interfaces of the Guided Mode are designed to logically guide users through the curation steps and include all necessary information such that no prior knowledge of the SPARC data standards is required. Contrary to the Free Form Mode, the Guided Mode interfaces are interconnected and form a single workflow such the curation process is streamlined further.

Begin curating a new dataset

Welcome to Guided Mode!

Guided Mode is designed to be the simplest and most efficient method of curating a SPARC dataset if you are starting from scratch. If you're not familiar with SPARC dataset curation standards, no worries! We will guide you step-by-step into curating your dataset according to the SPARC data standards. The user interfaces are designed to be very intuitive and will typically require you to only provide basic inputs or answer simple questions about your dataset.

Guided Mode is broken down into four main sections described below. During the first three sections, you will be guided to specify your data files locations and provide information about your dataset (i.e. metadata). Then, that information will be used in the fourth section by SODA to generate your dataset with the right folder structure and metadata files directly on Pennsieve. Note that none of your local data files will be ever modified or moved.

In this section, you will be asked information about the subjects and samples in your study. You will also be guided into specifying your data files locations from your computer such that they are included in the correct SPARC folders when the dataset is generated in section four of this process.

All SPARC datasets must include several metadata files, each containing various information that describes the content of the dataset. These include information such as the contributors that were involved in collecting/processing/curating the data, or the primary conclusion of the study. You will be guided in this section to provide all the required metadata. SODA will use that information to automatically create and include the required SPARC metadata files in your dataset when it is generated in section four of this process.

All SPARC datasets need to be uploaded on Pennsieve, a cloud-based data platform. For each of them, specific metadata is required on Pennsieve (such as a banner image or your dataset's description). This metadata will then be carried over to the SPARC Data Portal when your dataset becomes publicly available (this is handled outside the Guided Mode). In this section you will be guided to connect your Pennsieve account with SODA and provide all the required Pennsieve metadata that will be added to your Pennsieve dataset when it is generated in section four of this process.

In this final section, SODA will make use of all the information you have provided in the previous three sections to generate and upload you data and metadata on Pennsieve. Once generated, you will be able to see your dataset under your account on the Pennsieve web platform. Note that the dataset will only be visible to you and anyone you have provided permission to. Only after it has been reviewed by the Curation team and eventually made public by the PI of the dataset it will become publicly available on the SPARC Data Portal (this is not handled in the Guided Mode).

SPARC data is typically collected from subjects (either human or non-human) and/or from samples collected from such subjects (e.g., tissue samples). Code data

In the SDS, code data refers to any code related folders and files used in the generation or analysis of other data in your dataset.

Protocol data

In the SDS, protocol data refers to supplementary folders and files to accompany the experimental protocols that must be submitted to protocols.io (more on that at a later step).

Docs data

In the SDS, docs data refers to any data to be included in your dataset that does not belong to any of the other five categories (primary, source, derivative, code, or protocol data).

Manifest files

The SPARC guidelines require datasets to contain manifest files for each high level folder in a dataset. Manifest files should be placed directly in the high level folder, and contain information about all of the data files in the high level folder. The fields "filename", "timestamp", and "filetype" are mandatory fields required by the SPARC standards.

If you do not have manifest files, SODA can generate them for you, and they will be included in your dataset when it is generated in the last step of the guided workflow.

README information

All SPARC datasets must include a README file. This is a plain text file that contains necessary details for the reuse of the data beyond that which is captured in the other metadata file. Some information that should be included is:

  • How would a user use the files that are provided? e.g., first open file X and then look at file Y.
  • What additional details do users need to know? Are some subjects missing data?
  • Are there warnings about how to use the data or code?
  • Are there appropriate/inappropriate uses for this data?
  • Are there other places that users can go for more information? e.g., did you provide a GitHub repository or are there additional papers beyond what was provided in the metadata form?

You can provide that information below. SODA will use that information to automatically create the README metadata file and include it in your dataset when it is generated in the last step of this guided workflow.

Dataset permissions

Dataset permissions control which users or teams can view, edit, or manage your dataset on Pennsieve. Read below for a breakdown of the different types of dataset permissions:

  • Managers can invite people and teams to a dataset, edit and delete models, records and files, but cannot publish a dataset.
  • Editors can create, edit, or delete records and files in a dataset.
  • Viewers can view and browse all information in a dataset, but have no other permissions.
